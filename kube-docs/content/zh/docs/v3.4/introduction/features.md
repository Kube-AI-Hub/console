## 概览

Kube AI Hub 是面向 GPU/AI 算力场景的异构算力管理平台，提供从硬件接入到业务交付的完整管理链路。平台以 Kubernetes 为内核，通过统一的 Web 控制台提供可视化运维能力，涵盖节点管理、工作负载、存储、网络、监控告警、日志收集、多租户权限与计量计费等核心模块。

## 概览仪表板

集群概览页汇聚以下核心信息：

- 集群 CPU、内存、磁盘整体用量
- GPU 卡数量与实时利用率
- 节点健康状态（正常 / 异常 / 未就绪）
- 容器组运行状态（运行中 / 等待中 / 异常）
- 服务组件健康检查

## 节点管理

### 集群节点

展示所有计算节点的状态信息，支持对节点进行封锁（Cordon）、清空（Drain）和标签管理操作。节点详情页提供 CPU、内存、Pod 资源的实时监控图表及事件记录。

### 边缘节点

通过 KubeEdge 将算力调度延伸到边缘设备，支持云边协同的推理任务分发与管理。

### GPU 卡

列出集群中所有 GPU 卡，展示每张卡的型号、驱动版本、显存大小、利用率和温度等关键指标，支持 NVIDIA、华为昇腾、寒武纪、天数智芯等异构 GPU。

## 服务组件

展示 Kubernetes 核心服务组件（API Server、调度器、控制器、etcd 等）的健康状态与运行指标，帮助运维人员快速定位组件级故障。

## 项目（命名空间）管理

在集群层面统一管理所有项目（Namespace），提供项目级的资源配额设置、成员权限管理和资源用量概览。

## 应用负载

### 工作负载

管理集群级别的工作负载，包括：

- **部署（Deployment）**：无状态应用管理，支持滚动升级、回滚与弹性伸缩（HPA）
- **有状态副本集（StatefulSet）**：有状态服务管理，保障 Pod 的稳定标识与持久化存储
- **守护进程集（DaemonSet）**：确保每个节点运行指定 Pod，适用于节点监控和日志采集

工作负载详情页提供容器列表、环境变量、挂载卷、监控图表和事件记录，支持通过可视化界面编辑容器规格。

### 任务与定时任务

- **任务（Job）**：管理一次性批处理任务，支持并行执行与完成策略配置
- **定时任务（CronJob）**：基于 Cron 表达式定期触发的批处理任务，适用于数据清洗、定期报告等场景

### 容器组（Pod）

在集群级别跨所有命名空间查看容器组，支持按节点、状态、标签等条件筛选，可直接进入容器终端执行命令。

### 服务（Service）

管理集群内的 ClusterIP、NodePort、LoadBalancer 等类型的 Kubernetes 服务，查看端口映射与后端 Endpoint 状态。

### 路由（Ingress）

管理集群级别的 Ingress 规则，支持基于域名和路径的 HTTP/HTTPS 路由配置。

## 配置管理

- **密钥（Secret）**：安全存储敏感信息（密码、证书、Token），通过 RBAC 控制访问权限
- **配置字典（ConfigMap）**：管理应用配置，支持键值对和文件两种格式
- **服务账户（ServiceAccount）**：管理 Pod 的 API 访问凭证

## 存储管理

- **持久卷声明（PVC）**：声明持久化存储需求，支持动态供给（StorageClass）
- **存储类（StorageClass）**：定义存储后端策略，支持 Ceph、NFS、LocalPV 等多种存储方案
- **卷快照（Volume Snapshot）**：为持久卷创建时间点快照，支持快速恢复
- **快照类（VolumeSnapshotClass）**：定义快照驱动与删除策略

## 网络管理

- **网络策略（NetworkPolicy）**：定义 Pod 间通信规则，实现网络层隔离
- **容器组 IP 池（IPPool）**：管理 Pod IP 地址分配范围，支持与 Calico 集成
- **网关（Gateway）**：管理集群或项目级入口网关，提供统一外部访问入口

## 自定义资源（CRD）

面向平台管理员，提供集群内所有自定义资源定义（CRD）的可视化管理入口，支持查看自定义资源实例详情。

## 监控与告警

### 集群状态监控

提供秒级精度的集群级监控指标，包括：

- CPU 利用率、内存利用率、平均负载
- 磁盘使用量、I/O 吞吐、网络流量
- etcd 健康状态与 API Server 请求率

支持自定义时间范围查询与排行分析。

### GPU 资源监控

专为 GPU 算力场景设计的监控视图，展示每个节点和每张 GPU 卡的实时指标：

- GPU 利用率与显存占用率
- GPU 温度与功耗
- vGPU 虚拟化切分状态
- 多 GPU 类型（英伟达、昇腾、寒武纪等）统一呈现

### 应用资源监控

以工作负载和项目为维度展示资源用量，支持按 CPU、内存、网络等指标排行，快速定位资源消耗异常的工作负载。

### 告警消息

实时展示集群内触发的告警，包括节点异常、Pod 崩溃、存储告警等，支持按严重级别（紧急、警告、提示）筛选，并记录告警历史。

### 告警策略

自定义告警规则，针对节点、工作负载、Pod 等资源设置指标阈值与触发条件，支持配置告警重复间隔和通知接收人，通知渠道支持邮件、企业微信、钉钉、Slack 等。

## 日志收集

在集群级别配置统一的日志收集策略，支持对接以下日志存储后端：

- Elasticsearch
- Kafka
- Fluentd

支持多租户日志隔离，不同租户仅能查看属于自己的日志信息。

## 集群设置

- **基本信息**：查看集群名称、Kubernetes 版本、节点数量等基本信息
- **可见性**：控制集群对哪些企业空间开放访问
- **成员管理**：管理集群级别的用户与角色授权
- **角色管理**：自定义集群级别的访问角色与权限集合

## 多租户权限体系

Kube AI Hub 提供**平台 → 企业空间 → 项目**三级权限隔离体系：

- 平台管理员负责集群资源的统一管控
- 企业空间管理员负责资源配额与成员管理
- 项目成员在隔离的命名空间内独立开展工作

支持与 LDAP、Active Directory 集成，以及 SSO 单点登录。
