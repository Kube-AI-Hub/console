## 入口

点击页面右上角头像，在下拉菜单中选择 **新建应用空间**，进入创建表单页面。

## 填写表单

创建应用空间时，需要填写以下参数：

| 参数 | 是否必填 | 说明 |
|------|----------|------|
| 所有者 | 是 | 默认为当前登录用户；如果已加入组织，可切换为组织名 |
| 应用空间英文名 | 是 | 长度 2–64 个字符，以字母开头、字母或数字结尾，仅支持字母、数字、`-`、`_`、`.`，且不能连续使用 `-`、`_`、`.` |
| 别名 | 否 | 可选的中文名或友好名称 |
| 许可证 | 是 | 从下拉列表中选择合适的开源许可证 |
| 描述 | 否 | 简要描述应用空间的用途 |
| 最小副本数 | 是 | `0`：空间空闲时自动休眠以节省资源；`1`：空间始终保持运行 |
| SDK 类型 | 是 | 选择应用运行框架，详见下方说明 |
| Docker 模板 | 仅 Docker 类型 | 选择 Docker SDK 后显示，从平台预置模板中选择 |
| 集群 | 是 | 选择部署的计算集群 |
| 算力资源 | 是 | 根据所选集群，选择具体的计算资源规格（CPU/GPU） |
| 驱动版本 | 仅 GPU + Gradio/Streamlit | 选择 GPU 算力且 SDK 为 Gradio 或 Streamlit 时显示；可选 `11.8.0` 或 `12.1.0` |
| 环境变量 | 否 | 可选的键值对配置，支持普通变量和加密变量（Secret） |
| 可见性 | 是 | **公开**：所有用户可见；**私有**：仅本人和组织成员可见 |

### SDK 类型说明

| SDK 类型 | 说明 |
|----------|------|
| Gradio | 适用于快速构建机器学习演示界面 |
| Streamlit | 适用于构建数据应用和仪表板 |
| Docker | 适用于部署任意容器化 Web 应用，可选择平台预置模板 |
| Nginx | 适用于部署静态网站（仅管理员可选） |
| MCP Server | 适用于部署 MCP 协议服务 |

填写完成后，点击 **创建应用空间** 按钮提交。

## 初始化应用

创建成功后，平台会自动跳转至应用空间详情页。需要根据所选 SDK 类型推送对应的应用代码来初始化空间：

- **Gradio**：推送 `requirements.txt` 和 `app.py` 文件
- **Streamlit**：推送 `app.py` 文件
- **Docker**：推送 `Dockerfile` 和相关应用代码
- **MCP Server**：推送 MCP 服务配置和代码

{{< notice note >}}
不同 SDK 类型的初始化方式不同，请参考对应的应用类型文档了解详细步骤。
{{</ notice >}}
