## 编辑模型文件

### 本地编辑

通过 Git 或 SDK 将仓库克隆到本地，编辑文件后提交并推送。

```bash
# 克隆模型仓库
git clone https://<平台地址>/<命名空间>/<模型名称>
cd <模型名称>

# 编辑文件
# （对文件进行修改）

# 提交并推送
git add .
git commit -m "更新模型文件"
git push
```

如需身份验证，使用访问令牌：

```bash
git clone https://<用户名>:<访问令牌>@<平台地址>/<命名空间>/<模型名称>
```

### 在线编辑

平台支持直接在 Web 界面中编辑文本文件：

1. 进入模型仓库的 **文件** 标签页。
2. 点击需要编辑的文件（如 `README.md`、`config.json` 等文本文件）。
3. 点击文件内容区域右上角的 **编辑** 按钮，进入编辑模式。
4. 完成修改后，输入提交信息并点击 **提交** 按钮保存更改。

{{< notice note >}}
在线编辑仅支持文本文件。二进制文件（如模型权重文件）需要通过 Git 或 CLI 工具更新。
{{</ notice >}}

## 修改仓库设置

进入模型仓库的 **设置** 标签页，可以查看和修改仓库的基本信息。

### 可修改项

| 设置项 | 说明 |
|--------|------|
| 模型别名 | 模型的显示名称，可随时修改 |
| 模型简介 | 模型的描述信息 |
| 可见性 | 在 **公开** 和 **私有** 之间切换。修改为私有后，仅授权用户可访问 |
| 任务标签 | 设置模型的任务类型标签（如 `text-generation`、`image-classification` 等） |
| 行业标签 | 设置模型适用的行业分类标签 |

### 不可修改项

| 设置项 | 说明 |
|--------|------|
| 模型英文名称 | 模型仓库的唯一标识，创建后不可更改 |

{{< notice note >}}
修改任务标签或行业标签后，系统会自动更新 `README.md` 文件中的 YAML 元数据头。如果您已手动编辑过元数据，请注意标签变更可能会覆盖已有的元数据内容。
{{</ notice >}}

## 删除模型

如果不再需要某个模型仓库，可以将其删除。

1. 进入模型仓库的 **设置** 标签页。
2. 滚动到页面底部的 **危险操作** 区域。
3. 点击 **删除模型仓库** 按钮。
4. 在弹出的确认对话框中，输入模型的完整名称（格式为 `命名空间/模型名称`）以确认删除。
5. 确认后，模型仓库及其所有文件、提交记录将被永久删除。

{{< notice note >}}
删除操作不可逆，模型仓库中的所有数据（包括文件、提交历史和配置）将被永久删除且无法恢复。请在删除前确认已备份所有需要保留的文件。
{{</ notice >}}
