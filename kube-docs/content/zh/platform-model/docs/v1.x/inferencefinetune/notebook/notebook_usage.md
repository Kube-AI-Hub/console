## 启动 Notebook

创建完成并等待实例状态变为 **Running** 后，点击实例操作栏中的**启动 Notebook** 按钮，系统将在浏览器中打开对应的在线开发环境。

根据创建实例时选择的运行时框架，打开的开发环境类型不同：

| 运行时框架 | 开发环境说明 |
|------------|-------------|
| **JupyterLab** | 功能全面的交互式开发环境，支持 Notebook、终端、文件管理等 |
| **VS Code** | 基于 Web 的 VS Code 编辑器，提供完整的代码编辑和调试能力 |
| **Eclipse Theia** | 开源的云端 IDE，界面与 VS Code 类似，支持扩展插件 |

{{< notice note >}}
启动 Notebook 后，实例将持续占用算力资源并计费。请在使用完毕后及时停止实例。
{{</ notice >}}

## 查看运行日志

在实例列表中，点击实例名称进入实例详情页，切换到**分析**页签，可查看实例运行时的实时日志。

日志功能支持以下操作：

- **实时刷新**：自动滚动显示最新日志
- **关键字搜索**：快速定位特定日志信息
- **日志级别筛选**：过滤不同级别的日志输出（INFO、WARNING、ERROR 等）

日志内容包括环境启动信息、依赖安装记录、代码执行输出等，有助于排查实例运行中的问题。

## 查看计费详情

在实例详情页中切换到**计费**页签，可查看资源使用详情：

| 字段 | 说明 |
|------|------|
| **计费开始时间** | 实例启动并开始占用算力的时间 |
| **计费结束时间** | 实例停止或释放资源的时间 |
| **资源规格** | 当前实例使用的 GPU/CPU/内存配置 |
| **累计费用** | 截至当前的资源使用费用 |

{{< notice tip >}}
建议定期检查计费页面，及时了解资源消耗情况，避免不必要的费用。
{{</ notice >}}

## 停止实例

当开发工作暂时中断或完成后，点击实例操作栏中的**停止**按钮可暂停实例运行。

- 停止后实例不再占用算力资源，**计费暂停**。
- 实例中的数据和配置将被保留，下次启动后可继续使用。
- 停止操作不会删除实例，可随时重新启动。

## 删除实例

如果不再需要某个开发实例，点击实例操作栏中的**删除**按钮可永久删除实例。

{{< notice warning >}}
删除操作不可恢复。实例删除后，其中保存的代码、数据和配置将被永久清除。请在删除前确认已备份重要文件。
{{</ notice >}}
